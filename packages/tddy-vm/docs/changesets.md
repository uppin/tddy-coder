# Changesets Applied

Wrapped changeset history for tddy-vm.

**Merge hygiene:** [Changelog merge hygiene](../../../docs/dev/guides/changelog-merge-hygiene.md) — prepend one single-line bullet; do not rewrite shipped lines.

- **2026-07-02** [Feature] **VM & Image Library** — new `library.rs`: `VmLibrary` organizes `images/01-base/` (immutable, downloaded bases), `images/02-prepared-base/` (cloud-init-baked prepared bases, both chmod `0o444` via `set_readonly_file`), and `vm/<name>/` (per-VM `manifest.yaml` + mutable overlay + SSH keys) under the existing tddy data dir — no new env var; `init`/`import_base_image`/`write_manifest`/`read_manifest`/`list_manifests`/`remove_vm`/`create_vm` (async, real `qemu-img create` via an **absolute**-backing-path `vm_overlay_create_argv`, contrast `cloud_init::overlay_create_argv`'s co-located relative basename). New `vm_manifest.rs`: `VmManifest`/`RunPolicy`/`LoginPolicy` (YAML). `cloud_init.rs` gains `cloud_init_library_paths`, a pure path-mapping helper routing a cloud-init build's downloaded input base and its flattened-base+overlay pair into the library. `registry.rs`: `VmManager` gains a `Storage` enum and `from_library` constructor — per-VM manifests become the source of truth for library-backed managers (the pre-existing JSON-backed `VmManager::new` path is untouched, byte-for-byte). Design mirrors `~/Code/makers-lt`'s `maker-vm` qcow2-backing-file chaining; no image ever downloaded by this feature — tests reuse an already-built base supplied via `TDDY_CLOUDINIT_BASE_IMAGE`. 65 unit/acceptance tests pass (3 correctly-gated `#[ignore]`d, one run for real against makers-lt's `debian-12-base.qcow2`), `clippy -D warnings` clean. Feature [vm/tddy-vm.md](../../../docs/ft/vm/tddy-vm.md) § VM & Image Library; cross-package [changesets.md](../../../docs/dev/changesets.md). (tddy-vm, tddy-vm-build, tddy-daemon)

- **2026-07-02** [Feature] **`cloud_init` module — cloud-init image build with image-chaining** — new `packages/tddy-vm/src/cloud_init.rs`: pure argv/document builders (`base_convert_argv`, `overlay_create_argv`, `render_user_data`, `render_meta_data`, `completion_token`, `seed_iso_argv`, `iso_tool_command`, `cloud_init_boot_argv`, `classify_serial_line`) plus the async `build_cloud_init_image` orchestrator (copy base → convert to immutable qcow2 → resolve/generate SSH keypair → render+write NoCloud seed → build cidata ISO via `xorriso`/`mkisofs`/`genisoimage` → create the chained delta overlay → boot QEMU and watch the serial console for a `sha256`-derived completion token, with a durable `<name>-boot.log` transcript and a monitor-socket graceful-shutdown timeout fallback). Reuses `QemuVmArgs::monitor_socket_path`/`send_monitor_command` and mirrors `build.rs`'s progress-reporting and `tokio::select!` line-draining idioms. 22 unit tests pass; the real-boot tests are production tests — `#[ignore]`+`#[serial]` and additionally gated on `TDDY_CLOUDINIT_BASE_IMAGE` (no bundled/auto-discovered image; never run, even with `--ignored`, unless a developer explicitly supplies that env var) — verified to succeed end-to-end multiple times in a nested-virtualization sandbox when supplied (found+fixed a real `set_hostname`-misclassification bug and a `ds=nocloud` boot-speed fix along the way), though still timing-sensitive under that sandbox's fixed budgets. Feature [vm/tddy-vm.md](../../../docs/ft/vm/tddy-vm.md) § Cloud-init image build with image-chaining; cross-package [changesets.md](../../../docs/dev/changesets.md). (tddy-vm, tddy-vm-build)

- **2026-07-02** [Feature] **Image builder core + Docker toolchain for macOS** — `build.rs`: `ImageFormat`/`build_image(spec, output, format, progress)` extracted as the shared core behind both `tddy-vm-build`'s CLI and the existing `BuildVmImage` RPC (`build_vm_image_from_spec` refactored to call it, no behavior change — all 26 pre-existing tests still pass); macOS builds now route `make` through a Linux Docker container (`docker/buildroot-host/Dockerfile`, `HostToolchain`) since Buildroot rejects Apple Clang's `gcc` trampoline — required six sequential bugfixes (empty Nix-store bind mount, `$TMPDIR` sharing, missing CA certs + a stale-image-cache bug, a named-volume `/build` fixing a pkg-config exec-bit permission error, container-actual `nproc` for `-jN`, and injecting the written `.config` into that named volume before `olddefconfig` — found only after a real build silently ignored the caller's spec). Verified end-to-end against two real, non-mocked Buildroot builds (`#[ignore]`+`#[serial]`, ~62 min total). Feature [vm/tddy-vm.md](../../../docs/ft/vm/tddy-vm.md) § Image builder CLI; cross-package [changesets.md](../../../docs/dev/changesets.md). (tddy-vm, tddy-vm-build, tddy-sandbox-qemu, tddy-daemon)
- **2026-06-24** [Feature] **Long-running background Tasks — VM build fold-in** — `build.rs`: `VmBuildTaskBody` implements `TaskBody` with cooperative cancel via `CancellationToken` + SIGINT to `make` PID; `build_vm_image_from_spec` returns `bool` (true=success) mapping correctly to `Completed`/`Failed`/`Cancelled`; `service.rs`: `VmServiceImpl::new` accepts shared `TaskRegistry`; `BuildVmImage` spawns `VmBuildTaskBody` and streams its progress channel back as `BuildVmImageProgress`. Tests: 2 fold-in acceptance tests. Feature [daemon/background-tasks.md](../../../docs/ft/daemon/background-tasks.md). (tddy-vm)
