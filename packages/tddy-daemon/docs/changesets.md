# Changesets Applied

Wrapped changeset history for tddy-daemon.

- **2026-04-04** [Feature] Project **`main_branch_ref`** — **`ProjectData`** optional **`main_branch_ref`**; **`effective_integration_base_ref_for_project`**; **`add_project`** validates integration base strings via **`tddy_core::validate_integration_base_ref`** before YAML write. Feature doc: [git-integration-base-ref.md](../../../docs/ft/coder/git-integration-base-ref.md); [project-concept.md](../../../docs/ft/daemon/project-concept.md). (tddy-daemon)
- **2026-04-03** [Feature] Telegram notifier (teloxide) — Optional **`telegram`** YAML (**`enabled`**, **`bot_token`**, **`chat_ids`**); **`telegram_notifier`**: **`TelegramSessionWatcher`**, **`TelegramSender`**, **`send_telegram_via_teloxide`**, two-segment session labels, token masking; unit and integration tests with mock sender. Feature doc **[telegram-notifications.md](../../../docs/ft/daemon/telegram-notifications.md)**; package **[telegram-notifier.md](./telegram-notifier.md)**. (tddy-daemon)
- **2026-04-03** [Feature] Session workflow files + connection path alignment — **`ListSessionWorkflowFiles`** / **`ReadSessionWorkflowFile`** (**`session_workflow_files`**, **`session_workflow_files_rpc`**). **`sessions_base_for_user`** resolves the Tddy data root (**`~/.tddy`**) so **`unified_session_dir_path`** matches **`tddy-coder`** session trees. **`DeleteSession`** terminates a live recorded PID (SIGTERM/SIGKILL; Linux zombie handling), removes directories missing **`.session.yaml`** when safe, and exposes delete on active sessions from the web client. Web: **`SessionWorkflowFilesModal`**, **`SessionMoreActionsMenu`**, project/worktree matching (**`sessionProjectTable`**). Feature docs: [web-terminal.md](../../../docs/ft/web/web-terminal.md), [daemon changelog](../../../docs/ft/daemon/changelog.md), [connection-service.md](./connection-service.md). (tddy-daemon, tddy-service, tddy-web)
- **2026-03-29** [Feature] **`ListAgents`** and **`allowed_agents`** — Connection API lists allowed backends from YAML; **`StartSession`** rejects unknown **`agent`** when the allowlist is non-empty; label rules shared via **`agent_list_mapping`**. Feature docs: [web-terminal.md](../../../docs/ft/web/web-terminal.md), [local-web-dev.md](../../../docs/ft/web/local-web-dev.md), [systemd-install.md](../../../docs/ft/daemon/systemd-install.md). [connection-service.md](./connection-service.md). (tddy-daemon, tddy-service, tddy-web)
- **2026-03-28** [Feature] Session id validation on connection RPCs — **`ConnectSession`**, **`ResumeSession`**, and **`SignalSession`** call **`validate_session_id_segment`** before **`unified_session_dir_path`**; aligns with **`session_deletion`** for **`DeleteSession`**. [connection-service.md](./connection-service.md), [session-layout.md](../../../docs/ft/coder/session-layout.md). (tddy-daemon)
- **2026-03-28** [Feature] Multi-host daemon selection + ListSessions workflow enrichment — `ConnectionService`: `ListEligibleDaemons` via `EligibleDaemonSource`; `ListSessions` populates `daemon_instance_id` and enriched workflow fields (`session_list_enrichment`); `StartSession` honors local vs non-local `daemon_instance_id` (non-local returns unimplemented until peer routing); `spawn_blocking_with_timeout` for read + enrichment. Integration tests: `list_sessions_enriched`, multi-host. Feature docs: [connection-service.md](./connection-service.md), [docs/ft/web/web-terminal.md](../../../docs/ft/web/web-terminal.md). (tddy-daemon, tddy-service, tddy-web)
