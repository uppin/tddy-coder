# Changesets Applied

Wrapped changeset history for tddy-livekit.

**Merge hygiene:** [Changelog merge hygiene](../../../docs/dev/guides/changelog-merge-hygiene.md) — prepend one single-line bullet; do not rewrite shipped lines.

- **2026-07-23** [Feature] **Wire chunking codec into the RPC transport** — the `chunking.rs` codec shipped unwired in PR #322, so oversized LiveKit data-channel frames (>64,000-byte SCTP max) still wedged `publish_data` (`SharedPublisher` retried 30× forever), starving small RPCs. Now wired end-to-end: send frames via `frame_for_transport`/`next_message_id` (`client.rs::publish_request`, both `participant.rs` response drains), receive reassembles per sender via `ChunkReassembler` (`client.rs`, `client_factory.rs`, `participant.rs`). Small payloads stay raw (byte-for-byte); only oversized ones split, each chunk frame prefixed with a `0x00` magic byte a valid prost `RpcRequest`/`RpcResponse` can never begin with, so peers disambiguate raw vs chunk on the first byte (`is_chunk_frame`). TS mirror in `tddy-livekit-web/src/chunking.ts`. Verified by a real-LiveKit `rpc_scenarios` 370 KB oversized-echo round-trip. Feature [rpc-multi-transport.md § LiveKit oversized-frame chunking](../../../docs/ft/coder/rpc-multi-transport.md#livekit-oversized-frame-chunking). Cross-package [docs/dev/changesets.md](../../../docs/dev/changesets.md). PR [#323](https://github.com/uppin/tddy-coder/pull/323). (tddy-livekit, tddy-livekit-web)
- **2026-07-12** [Docs] **`session` participant metadata key** — documents the `session` metadata block published by `tddy-coder`'s participant (sibling of `owned_project_count` / `codex_oauth`, shallow-merged via `merge_participant_metadata_json`): `session_id`, `workflow_goal`, `workflow_state`, `elapsed_display`, `agent`, `model`, `activity_status`, `recipe`, `repo_path`, `pending_elicitation`. No new public API; the merge helper already existed. `participant_metadata_unit` extended to assert the `session` key is preserved across merges. Feature [participant-metadata.md § `session` metadata key](participant-metadata.md#session-metadata-key). Cross-package [docs/dev/changesets.md](../../../docs/dev/changesets.md). PR [#297](https://github.com/uppin/tddy-coder/pull/297). (tddy-livekit)
- **2026-07-01** [Refactor] **`RpcClient`/`LiveKitParticipant` onto shared `tddy-rpc` engines** — `RpcClient` now holds `Arc<ClientEngine>` (was hand-rolled pending-request maps) and implements `RpcClientTransport`; `LiveKitParticipant` now holds `Arc<ServerEngine<S>>` (removed `ActiveStream`/`BidiStreamMeta`/`BidiSession`/`SessionKey`/`spawn_streaming_response`, replaced by `spawn_response_drain`/`spawn_response_drain_reconnectable`). `ServerEngine` needed zero changes — LiveKit-specific concerns (multi-remote identity resolution, pending-data buffering, `SharedPublisher` reconnect-resilient publishing) live around `on_request`, not inside it. Envelope moved to `tddy_rpc::envelope` (own `proto/rpc_envelope.proto` deleted; `build.rs` no longer compiles it). Wire format and TypeScript client compatibility unchanged. Behavior fix: single-message bidi calls now correctly route through `start_bidi_stream` (previously fell through to `handle_rpc_stream`'s default single-message path). Feature [rpc-multi-transport.md](../../../docs/ft/coder/rpc-multi-transport.md). (tddy-livekit)
