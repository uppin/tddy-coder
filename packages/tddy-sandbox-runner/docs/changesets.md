# Changesets Applied

Wrapped changeset history for tddy-sandbox-runner.

**Merge hygiene:** [Changelog merge hygiene](../../../docs/dev/guides/changelog-merge-hygiene.md) — prepend one single-line bullet; do not rewrite shipped lines.

- **2026-07-06** [Feature] **`--agent-kind cursor` PTY spawn** — `spawn_cursor_pty` invokes `node index.js` from the resolved `agent` bundle; `SandboxRunnerArgs.agent_kind` selects claude vs cursor recipe/MCP argv. Feature [cursor-cli-session.md](../../../docs/ft/daemon/cursor-cli-session.md). Cross-package: [docs/dev/changesets.md](../../../docs/dev/changesets.md). PR [#287](https://github.com/uppin/tddy-coder/pull/287). (tddy-sandbox-runner, tddy-sandbox-recipes, tddy-daemon)
- **2026-07-03** [Feature] **`--claude-arg` pass-through + plain-HTTP forward proxy + persisted MCP logs** — new repeated `--claude-arg` (`allow_hyphen_values`) appended verbatim to the in-jail `claude` argv after the fixed flags + before the MCP args (the MCP block's trailing `--mcp-config` is variadic and would otherwise swallow a trailing positional prompt); ignored in `--pty-command` mode. Egress shim gained a forward-proxy path: `rewrite_http_proxy_request` (absolute-form → origin-form, extracts host:port) + `handle_http_forward` (relay tunnel; host owns the outbound socket — no jail net rule needed); CONNECT handler refactored to share `open_relay_tunnel` + `pump_tunnel`, so `base_url: http://localhost:11434` works as-is. `write_claude_mcp_config` now writes a `tddy-tools --mcp` `env` block; the runner sets `TDDY_TOOLS_LOG_FILE` → `<session-dir>/egress/tddy-tools.mcp.log` and `RUST_LOG` (default `info,tddy_tools=debug,tddy_discovery=debug`, override via `mcp_log_level` config / `--mcp-log-level`); app maintains a `<session-base>/sessions/latest` symlink. Feature [managed-codebase-subagents.md § Standalone launcher](../../../docs/ft/coder/managed-codebase-subagents.md#standalone-launcher-claude-sandbox). (tddy-sandbox-runner, tddy-sandbox-app, tddy-tools, tddy-discovery)
- **2026-07-03** [Feature] **`--append-system-prompt-file` passthrough** — `SandboxRunnerArgs` gains `append_system_prompt_file`, threaded through `SpawnClaudePtyParams` into the in-jail `claude` argv, so a daemon-hosted managed-codebase sandboxed session can inject the workflow recipe's orchestration system prompt. Feature [managed-codebase-workflow.md](../../../docs/ft/coder/managed-codebase-workflow.md). Cross-package: [docs/dev/changesets.md](../../../docs/dev/changesets.md). (tddy-sandbox-runner, tddy-daemon)
- **2026-07-02** [Feature] **`resolve_subagent_replaced_tools`/`TDDY_SUBAGENT_REPLACES` — allowlist filtering + drift guard** — `spawn_claude_pty` now resolves the effective replaced-tool set (env override or the subagent's declared default) and threads it into `append_claude_mcp_args`; new drift-guard test round-trips every `workspace_exec_tool_names()` entry through `tddy_discovery::resolve_replaced_tools` to catch the two crates' tool-name tables falling out of sync. Feature [managed-codebase-subagents.md § Tool replacement](../../../docs/ft/coder/managed-codebase-subagents.md#tool-replacement-subagent-declared). (tddy-sandbox-runner, tddy-discovery, tddy-sandbox)
- **2026-07-02** [Feature] **`grpc_socket` arg made optional** — `SandboxRunnerArgs::grpc_socket` is now `Option<PathBuf>`; it was already vestigial (unused once `--stdio` is passed, never read anywhere else in the crate), but was still a required flag, which blocked `tddy-daemon` from spawning the runner with `--stdio` and no gRPC flags at all. Cross-package: [docs/dev/changesets.md](../../../docs/dev/changesets.md). (tddy-sandbox-runner, tddy-daemon)
- **2026-07-02** [Feature] **`--allowedTools` reflects whether a discovery subagent is configured** — `spawn_claude_pty` reads `TDDY_SUBAGENT` from its own process env and passes the resulting bool to `append_claude_mcp_args` (now takes `subagent_enabled: bool`), so a sandboxed Claude's allowlist actually includes `mcp__tddy-tools__subagent_*` end-to-end when `tddy-sandbox-app --discovery-subagent` was given. Feature [managed-codebase-subagents.md](../../../docs/ft/coder/managed-codebase-subagents.md). (tddy-sandbox-runner, tddy-sandbox-recipes, tddy-sandbox-app)

- **2026-07-01** [Feature] **`--stdio` transport for `SandboxService`; `run_host_relay` made transport-agnostic** — `--stdio` flag serves `SandboxService`'s `RpcService` impl (`Echo`/`EchoStream`/`SessionChannel`, incl. real PTY output) over `StdioEndpoint`, as an alternative to `--grpc-uds`/`--grpc-listen-port` (not yet removed — still the only transport `tddy-daemon`'s real call sites use). New `SessionChannelClient` trait (`host_relay.rs`) abstracts the one seam where the tonic and stdio transports genuinely differ (a typed tonic bidi call vs. `tddy-rpc`'s untyped `call_unary`/`start_bidi_stream` interface — no generated client stub exists for RpcService-flavored services); implemented for the existing tonic `SandboxClient` (unchanged, purely additive) and a new `StdioSandboxClient`. `run_host_relay`'s ~130 lines of PTY/tool/tunnel/egress relay logic needed **no changes** — it already only touches plain `SessionFrame` structs, the same Rust type on both transports now that `sandbox.proto`'s message types are `extern_path`-unified across its tonic and RpcService codegen passes (a separate, pre-existing gap this surfaced and fixed). Proven end-to-end through a real macOS Seatbelt jail, including a full tool-call round trip dispatched via `tddy_tools::session_tool_client::dispatch_via_sandbox_ipc` — the same path a real MCP tool call uses. Feature [rpc-multi-transport.md](../../../docs/ft/coder/rpc-multi-transport.md); architecture [tddy-sandbox architecture](../../tddy-sandbox/docs/architecture.md#control-channel-transport). PR [#253](https://github.com/uppin/tddy-coder/pull/253). (tddy-sandbox-runner, tddy-sandbox, tddy-sandbox-darwin, tddy-daemon, tddy-service, tddy-stdio, tddy-rpc)
