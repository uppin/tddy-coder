## Packages changed

### `packages/tddy-vm` (new)
- New crate: `Vm` trait, `QemuVm`, `MockVm`, `QemuVmArgs`, `VmManager` (full impl with JSON persistence), `VmSpec`, `VmState`, `build_vm_image`, `VmServiceImpl`.
- `src/build.rs`: added `VmImageRecord`, `built_images_dir()`, `list_built_images_in()`, `list_built_images()` for directory scanning.
- `src/service.rs`: `SessionUserResolver` type (local, avoids circular dep); `VmServiceImpl::new` now requires resolver; `authenticate()` helper; `list_vm_images` handler; all handlers validate token before execution.
- `tests/list_vm_images_unit.rs`: 6 unit tests covering scan, sort, empty-dir, extension filtering.

### `packages/tddy-service`
- `proto/vm.proto`: new `VmService` with 8 RPCs (7 original + `ListVmImages`); new `ListVmImagesRequest`, `VmImageInfo`, `ListVmImagesResponse` messages.
- `build.rs`: codegen for `vm.proto` (TddyServiceGenerator + descriptor pass).
- `src/lib.rs`: `pub mod vm { include!(...) }`, `pub use proto::vm::VmServiceServer`.

### `packages/tddy-daemon`
- `Cargo.toml`: `tddy-vm` dep added.
- `src/main.rs`: `VmManager` + `VmServiceImpl` constructed with resolver cloned from auth; `vm.VmService` ServiceEntry pushed inside `if let Some(user_resolver)` block (gated on auth config, same as `ConnectionService`).
- `tests/vm_service_acceptance.rs`: updated to pass resolver; 5 new tests (auth rejection for ListVms/DefineVm/ListVmImages, ListVmImages success, ListVmImages auth failure).

### `packages/tddy-demo-runner`
- `src/vm.rs`, `src/qemu.rs`, `src/mock.rs` deleted.
- `Cargo.toml`: `tddy-vm` dep added; `serde`, `serde_json` removed.
- `src/lib.rs`: re-exports from `tddy_vm` instead of own modules.
- `src/orchestrator.rs`: uses `tddy_vm::{Vm, VmError, RunningVm, PortForward}`.
- `tests/`: imports updated to use `tddy_vm` types (`QemuVm`, `MockVm`, `VmConfig`, `PortForward`).

### `packages/tddy-daemon` (connection_service.rs)

- `start_demo_vm` / `stop_demo_vm`: uses `tddy_vm::QemuVm` + `tddy_vm::Vm` + `tddy_vm::VmConfig`.
